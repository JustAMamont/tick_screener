# High-Performance Real-Time Market Anomaly Scanner

Асинхронный сканер для мониторинга криптовалютных рынков в режиме реального времени. Система агрегирует потоки сделок (trades) через WebSockets, формирует кастомные временные интервалы (OHLCV) и детектирует аномальные всплески волатильности и объема.

## Проблематика

При разработке систем мониторинга рыночных данных на Python возникают три основные проблемы:

1. **CPU Bottleneck:** Обработка тысяч тиков в секунду по 500+ торговым парам в чистом Python вызывает значительные задержки из-за накладных расходов на интерпретацию кода и работу Garbage Collector.
2. **Memory Management:** Хранение истории сделок в стандартных списках (`list`) приводит к фрагментации памяти и высокой нагрузке на систему при постоянных операциях вставки/удаления.
3. **Network Latency & Stability:** Поддержание сотен WebSocket-соединений требует эффективного управления асинхронными задачами и механизмами быстрого восстановления сессий.

## Решение

Данный проект решает эти проблемы через гибридную архитектуру:

* **Вычисления на уровне C:** Критическая логика агрегации данных и расчета аномалий вынесена в **Cython**-модули. Использование типизированных структур C и прямого доступа к памяти (MemoryViews) исключает оверхед Python.
* **Оптимизация памяти:** Вместо хранения объектов Python, данные хранятся в кольцевых буферах на базе **NumPy** массивов, что обеспечивает константную сложность $O(1)$ для операций записи и предотвращает утечки памяти.
* **Сверхбыстрый Event Loop:** Использование **uvloop** (замена стандартного asyncio loop на базе libuv) позволяет минимизировать задержки при обработке сетевых событий.

## Основные характеристики

* **Custom Timeframes:** Генерация свечей любого произвольного размера (например, 1s, 5s, 15s), которые обычно не предоставляются биржами через стандартные API.
* **Volatility Detection:** Алгоритм отслеживает не только цену закрытия, но и полную внутрисвечную волатильность ($High-Low$), позволяя ловить резкие «прострелы» рынка.
* **Hot Reload:** Полная поддержка обновления конфигурации без прерывания активных WebSocket-сессий.
* **Listing Detector:** Автоматическое обнаружение новых торговых пар при обновлении рынков биржи.
* **Fault Tolerance:** Автоматический экспоненциальный backoff при сетевых ошибках и переподключение воркеров.

## Архитектура системы

Проект разделен на функциональные слои:

1. **Transport Layer (`ccxt.pro`)**: Агрегация сырых данных через WebSockets.
2. **Processing Layer (`Cython`)**:
    * `TimeWindowTrades`: C-класс для управления скользящим окном данных.
    * `check_alert`: Высокопроизводительный фильтр аномалий.
3. **Application Layer**: Управление воркерами, логирование (Lumina) и доставка уведомлений в Telegram.

## Параметры конфигурации

### Динамические настройки (`config.json`)

| Параметр | Описание |
| :--- | :--- |
| `return_limit` | Порог изменения цены (%) для срабатывания алерта. |
| `volume_limit` | Минимальный порог объема в котируемой валюте (USD). |
| `trange` | Окно агрегации данных в секундах. |
| `currency_type` | Режим рынка: `spot` или `swap` (бессрочные фьючерсы). |
| `blacklist` | Список пар, исключенных из мониторинга. |

### Аргументы инициализации (класс `Main`)

* `multi_symbol_chunk_size` (default: 40): Количество пар на одно WS-соединение (для бирж с поддержкой batch-подписок).
* `single_symbol_chunk_size` (default: 100): Лимит задач на один процесс для бирж без batch-подписок.
* `task_launch_delay`: Пауза между запуском воркеров для предотвращения Rate Limit при старте.

## Сборка и запуск

### Системные требования

* Python 3.10+
* C-компилятор (gcc / clang / msvc) — *для сборки Cython-модулей.*
* **Rust Toolchain** ([rustup.rs](https://rustup.rs)) — *необходим для автоматической компиляции высокопроизводительного логгера Lumina из исходников при установке зависимостей.*

### Установка

1. Клонируйте репозиторий, создайте виртуальное окружение и установите зависимости:

   ```bash
   pip install -r requirements.txt
   ```

   *(Примечание: `requirements.txt` содержит `git+https://...` ссылку на логгер Lumina, он будет скачан и собран автоматически).*

2. Скомпилируйте Cython-модули с автоматической очисткой временных файлов:

   ```bash
   python setup.py build_ext --inplace
   ```

3. Настройте `config.json` (укажите ваш Telegram токен и настройки фильтрации).
4. Запустите сканер:

   ```bash
   python main.py
   ```

## Логирование и мониторинг (Lumina)

Для достижения максимальной производительности и исключения блокировок Event Loop-а, сканер использует кастомный гибридный логгер [Lumina](https://github.com/JustAMamont/lumina_project) (Rust Core).

Логи сохраняются в структурированном бинарном формате с компрессией (`.ldb`), поэтому для их чтения, фильтрации и анализа в реальном времени используется встроенная CLI-утилита `lumina`.

### Основные команды CLI

Убедитесь, что вы находитесь в корневой директории проекта (где лежит папка с логами), и используйте следующие команды:

* **Чтение всех логов** (автоматический поиск и парсинг файлов в директории):

  ```bash
  lumina
  ```

* **Режим Live (Tail)** — просмотр логов в реальном времени (аналог `tail -f`):

  ```bash
  lumina -f
  ```

* **Фильтрация по уровню** — показать только Warning, Error и Critical:

  ```bash
  lumina -wec
  ```

  *(Либо используйте `--min-level 30`)*
* **Поиск (Grep)** — найти логи по конкретной монете, событию или тексту:

  ```bash
  lumina --grep "BTC/USDT"
  ```

### Экспорт логов для аналитики

Если вам нужно проанализировать логи через сторонние инструменты (например, загрузить в базу данных или обработать через `jq`), их можно легко экспортировать в формат JSON Lines. Это позволяет фильтровать события по структурированным полям (kwargs), которые сканер передает в логгер.

* **Сохранить логи за последний день в файл:**

  ```bash
  lumina --days 1 --export-json scanner_logs.jsonl
  ```

* **Найти все события отвалов сети (ошибки соединения / DDoS-защита):**

  ```bash
  lumina --json -wec | jq '. | select(.context.network_exc != null)'
  ```

* **Отследить предупреждения о переполнении очереди для конкретной монеты:**

  ```bash
  lumina --json -w | jq '. | select(.context.symbol == "BTC/USDT")'
  ```

> **Примечание:** Утилита `lumina` устанавливается автоматически вместе с остальными python-зависимостями. Если терминал пишет, что команда не найдена, просто убедитесь, что вы **активировали виртуальное окружение** сканера.

## Разработка

Для модификации логики расчетов используйте файлы `.pyx`. После изменений требуется повторная компиляция через `setup.py`.

**Файловая структура:**

* `utils/scanner.pyx` — Асинхронная логика приложения и управление воркерами.
* `utils/cythonic_utils.pyx` — Низкоуровневая обработка данных и математика.
* `setup.py` — Кастомный скрипт сборки с автоматическим cleanup промежуточных `.c` файлов.
